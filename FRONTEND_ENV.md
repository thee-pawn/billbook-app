# Frontend Environment Variables Guide

## Overview

The Electron app automatically creates a `.env.production` file in your frontend repository. This file is **NOT committed to git** and contains production-specific environment variables.

---

## Automatic Variables

The app automatically creates these variables:

```bash
# Backend API URL
VITE_API_URL=http://localhost:4242

# App Configuration  
VITE_APP_NAME=BillBook Application
VITE_APP_ENV=production

# Ports (for reference)
VITE_BACKEND_PORT=4242
VITE_FRONTEND_PORT=3000
```

---

## Adding Custom Variables

### Method 1: Via `config.local.json` (Recommended)

Add a `frontendEnv` object to your `config.local.json`:

```json
{
  "githubToken": "ghp_...",
  "backendRepoUrl": "https://github.com/thee-pawn/whatsapp-automation.git",
  "frontendRepoUrl": "https://github.com/thee-pawn/billbook-frontend.git",
  "backendPort": 4242,
  "frontendPort": 3005,
  
  "frontendEnv": {
    "VITE_CUSTOM_API_KEY": "your-api-key-here",
    "VITE_FEATURE_FLAG": "true",
    "VITE_ANALYTICS_ID": "UA-123456",
    "VITE_ENVIRONMENT": "production"
  }
}
```

**Result:** These will be added to `.env.production`:

```bash
# Custom environment variables
VITE_CUSTOM_API_KEY=your-api-key-here
VITE_FEATURE_FLAG=true
VITE_ANALYTICS_ID=UA-123456
VITE_ENVIRONMENT=production
```

### Method 2: Manual Edit (After First Run)

After the first run, you can manually edit:
```
~/Library/Application Support/Electron/repos/frontend/.env.production
```

**Note:** Manual changes will be **overwritten** on next app restart.

## Using Environment Variables in Frontend

In your React/Vite app:

```typescript
// Access environment variables
const apiUrl = import.meta.env.VITE_API_URL;
const appName = import.meta.env.VITE_APP_NAME;
const customKey = import.meta.env.VITE_CUSTOM_API_KEY;

console.log('API URL:', apiUrl);
console.log('App Name:', appName);
console.log('Custom Key:', customKey);
```

---

## Important Notes

### 1. Variable Naming
- ✅ Must start with `VITE_` to be exposed to your frontend
- ❌ Variables without `VITE_` prefix won't be accessible
- ✅ Use UPPERCASE with underscores: `VITE_MY_VAR`

### 2. File Location
```
~/Library/Application Support/Electron/repos/frontend/.env.production
```

### 3. Regeneration
The `.env.production` file is **regenerated** every time you start the Electron app.

### 4. Git Ignore
Make sure `.env.production` is in your frontend's `.gitignore`:

```gitignore
# Frontend .gitignore
.env.local
.env.production
.env.*.local
```

## Examples

### Example 1: API Configuration

**config.local.json:**
```json
{
  "frontendEnv": {
    "VITE_API_URL": "http://localhost:4242",
    "VITE_API_TIMEOUT": "30000",
    "VITE_API_RETRY": "3"
  }
}
```

**Frontend usage:**
```typescript
const config = {
  apiUrl: import.meta.env.VITE_API_URL,
  timeout: parseInt(import.meta.env.VITE_API_TIMEOUT),
  retry: parseInt(import.meta.env.VITE_API_RETRY)
};
```

### Example 2: Feature Flags

**config.local.json:**
```json
{
  "frontendEnv": {
    "VITE_ENABLE_ANALYTICS": "true",
    "VITE_ENABLE_DEBUG": "false",
    "VITE_ENABLE_CHAT": "true"
  }
}
```

**Frontend usage:**
```typescript
const features = {
  analytics: import.meta.env.VITE_ENABLE_ANALYTICS === 'true',
  debug: import.meta.env.VITE_ENABLE_DEBUG === 'true',
  chat: import.meta.env.VITE_ENABLE_CHAT === 'true'
};

if (features.analytics) {
  initializeAnalytics();
}
```

### Example 3: Third-Party Services

**config.local.json:**
```json
{
  "frontendEnv": {
    "VITE_STRIPE_PUBLIC_KEY": "pk_test_...",
    "VITE_GOOGLE_MAPS_KEY": "AIza...",
    "VITE_SENTRY_DSN": "https://..."
  }
}
```

## Best Practices

1. ✅ **Use config.local.json** for environment variables
2. ✅ **Never commit secrets** - they're in local config only
3. ✅ **Prefix all variables with VITE_**
4. ✅ **Use meaningful names**: `VITE_API_BASE_URL` not `VITE_URL`
5. ✅ **Document your variables** in this file or README
6. ✅ **Keep sensitive data** (API keys, tokens) in `config.local.json`

## Troubleshooting

### Variable not available in frontend

**Problem:** `import.meta.env.VITE_MY_VAR` is `undefined`

**Solutions:**
1. Check variable starts with `VITE_`
2. Restart the Electron app (regenerates .env.production)
3. Check the generated file exists
4. Rebuild frontend: the app does this automatically

### Old values persist

**Problem:** Changed `config.local.json` but frontend uses old values

**Solution:**
1. Restart the Electron app completely
2. The `.env.production` will be regenerated
3. Frontend will be rebuilt with new values
